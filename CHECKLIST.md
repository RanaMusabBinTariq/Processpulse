# ProcessPulse Hireable 30-Point Checklist

Implemented in this build:

1. One selected issue across the app.
2. Selected issue ribbon on each operational page.
3. Action-first dashboard.
4. Next recommended action visible.
5. No clipped metric cards for important values; readable review ticket rows.
6. Reduced oversized cards/fonts and added responsive layout.
7. Removed empty visual bars.
8. Added Data -> Signal -> Workflow -> Evidence -> Requirements process-flow graphic.
9. Added manufacturing workflow graphic with affected step/status.
10. Demo scenario buttons with visible behavior changes.
11. Normal vs pH/flow/temperature/missing IPC before/after scenario behavior.
12. Beginner-friendly Data Intake with required columns and aliases.
13. Missing data is not faked or plotted as normal.
14. Signal/XAI page uses readable review ticket.
15. Plain-language explanations for SPC, Cpk, XAI, and OEE.
16. “What this page does” help box on each page.
17. Workflow/eBR reacts to selected issue.
18. Workflow actions update status/audit trail.
19. Audit trail shows timestamp, actor, and action.
20. Process Twin highlights selected/focus parameter.
21. Process Twin sliders visibly change risk, yield, quality, driver.
22. Sensors & OEE explains causes/estimation.
23. Requirements Brief is issue-specific and bilingual labels are provided.
24. Exportable evidence pack.
25. About page is recruiter-facing.
26. About page has no operational batch cards.
27. Candidate identity, LinkedIn, GitHub, email visible.
28. Strong legal disclaimer on every page and About.
29. English/German UI labels and recruiter-facing text.
30. App tells one story: data -> signal -> workflow -> twin/evidence -> requirements.
